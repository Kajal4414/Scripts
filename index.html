ASUS Configuration

Battery threshold settings and ASUS service management.

Set Battery Charging Threshold (60%)

Requires the ASUS System Control Interface driver. MyASUS app is not needed.

1

Command Prompt (Administrator)

cmd 
reg add "HKLM\SOFTWARE\ASUS\ASUS System Control Interface\AsusOptimization\ASUS Keyboard Hotkeys" /v ChargingRate /t REG_DWORD /d 60 /f
2

Registry Editor (Manual)

Navigate to the key path below, then set ChargingRate to 3C (hex) or 60 (dec).

HKLM\SOFTWARE\ASUS\ASUS System Control Interface\AsusOptimization\ASUS Keyboard Hotkeys
3

PowerShell Script (Alternative)

Use the local helper script at scripts/windows/charging_threshold.ps1.

ASUS Services Management

powershell 
Get-Service | Where-Object { $_.Name -like "*ASUS*" }

Safe to disable non-essential system services (keep ASUSOptimization active):

AsusAppService ASUSLinkNear ASUSLinkRemote ASUSSoftwareManager ASUSSwitch ASUSSystemAnalysis ASUSSystemDiagnosis
powershell 
"AsusAppService","ASUSLinkNear","ASUSLinkRemote","ASUSSoftwareManager","ASUSSwitch","ASUSSystemAnalysis","ASUSSystemDiagnosis" | ForEach-Object { Set-Service -Name $_ -StartupType Disabled }
powershell 
Restart-Service "ASUSOptimization"

Boot Cleanup

Remove unwanted recovery partitions and clean up dual-boot EFI residues.

Caution: Disk and partition operations are permanent. Back up files before execution. Run prompts as Administrator.

Remove Windows Recovery Partition

Removes the hidden recovery partition (~500 MB). Only do this if you have installation media.

1

Open elevated Command Prompt

Press Win + X → "Terminal (Admin)" or "Command Prompt (Admin)".

2

Select disk and partition inside diskpart

diskpart 
diskpart
list disk
select disk 0       # Replace 0 with appropriate disk number
list partition      # Note the 'Recovery' partition number
3

Delete recovery partition using override

diskpart 
select partition X    # Replace X with recovery partition number
delete partition override

Remove Linux Boot Entry (Dual Boot Cleanup)

Removes EFI GRUB residues after deleting Linux partitions.

1

Assign temporary letter to EFI partition

diskpart 
diskpart
list disk
select disk 0
list partition
select partition 1  # Select FAT32 System EFI partition (~100MB)
assign letter=R
exit
2

Remove directory corresponding to Linux boot

cmd 
R: && cd EFI && dir && rd ubuntu /s
3

Unassign the temporary drive letter

diskpart 
diskpart
select volume R
remove letter=R
exit

UEFI Repair

Backup and restore BCD configuration, or repair corrupted EFI Boot Partitions.

Always back up BCD config first. Rebuilding partitions deletes unrelated boot options.

Backup BCD Store

cmd 
bcdedit /export C:\BCD_Backup\bcdbackup

Restore BCD Store

cmd 
bcdedit /import C:\BCD_Backup\bcdbackup

Quick EFI Fix — Repair In Place

Use when BCD is corrupt but partition layout is intact. Reference: youtube.com/CZ17JrgFFhw.

1

Format and label EFI partition

diskpart 
diskpart
list disk
select disk 0
list volume
select partition 1    # Note the ~100MB FAT32 partition
format fs=fat32 quick
assign letter=V
exit
2

Rebuild boot parameters using bcdboot

cmd 
bcdboot C:\Windows /s V: /f UEFI
3

Unassign the helper drive letter

diskpart 
diskpart
select volume V
remove letter=V
exit

Full EFI Recreate — From Setup Command Line

Use when the EFI partition is missing or unrecoverable. Run inside a Windows installation PE (Press Shift + F10).

1

Delete the corrupted EFI partition

diskpart 
diskpart
list disk
select disk 0
list partition
select partition 1    # Note FAT32 EFI Partition number
delete partition
2

Recreate the System partition

diskpart 
create partition efi size=260
format quick fs=fat32 label="System"
assign letter=S
exit
3

Deploy boot files

cmd 
bcdboot C:\Windows /s S: /f UEFI

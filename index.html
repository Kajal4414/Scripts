ASUS Configuration

Battery threshold settings and ASUS service management.

Set Battery Charging Threshold (60%)

Requires the ASUS System Control Interface driver. Does not need the MyASUS app.

1

Command Prompt (Administrator)

cmd 
reg add "HKLM\SOFTWARE\ASUS\ASUS System Control Interface\AsusOptimization\ASUS Keyboard Hotkeys" /v ChargingRate /t REG_DWORD /d 60 /f
2

Registry Editor (Manual)

Navigate to the following key, then set ChargingRate to 3C (hex) or 60 (decimal).

HKLM\SOFTWARE\ASUS\ASUS System Control Interface\AsusOptimization\ASUS Keyboard Hotkeys
3

PowerShell Script (Alternative)

Use the pre-built script at ../Extra/charging_threshold.ps1.

Boot Cleanup

Remove unwanted recovery partitions and wipe Linux boot entries from the EFI partition after a dual-boot teardown.

Caution: Partition deletion is permanent and cannot be undone. Back up all important data first. Run every command from an elevated (Administrator) prompt.

Remove Windows Recovery Partition

The recovery partition (~500 MB, hidden) is used by Windows Reset. Once deleted, factory reset options that keep or remove files will stop working. Only remove it if you have a bootable Windows USB or ISO as a fallback.

1

Open Command Prompt as Administrator

Press Win + X → "Command Prompt (Admin)" or "Windows PowerShell (Admin)".

2

Launch Diskpart and list partitions

diskpart 
diskpart
list disk
select disk 0       # Replace 0 with the disk containing the recovery partition
list partition      # Find the "Recovery" type entry (~500 MB)
3

Select and delete the recovery partition

diskpart 
select partition X    # Replace X with the recovery partition number
delete partition override

The override flag is required — Windows normally protects this partition from deletion.

Remove Linux Boot Entry (Dual Boot)

After deleting the Linux partition, its EFI boot entry lingers on the EFI System Partition. This removes it so Windows boots directly without any GRUB menu.

1

Assign a temporary drive letter to the EFI partition

Open an elevated Command Prompt, then run:

diskpart 
diskpart
list disk
select disk 0       # Replace 0 with the correct disk
list partition
select partition 0  # Replace 0 with the EFI partition number (~100 MB FAT32)
assign letter=R
exit
2

Navigate to the EFI directory and remove the Linux entry

cmd 
R:
cd EFI
dir                  # List entries — look for ubuntu, fedora, arch, etc.
rd ubuntu /s         # Replace 'ubuntu' with the actual folder name

Type Y and press Enter when asked to confirm deletion.

3

Remove the temporary drive letter and reboot

diskpart 
diskpart
select volume R
remove letter=R
exit

Restart your system — GRUB will no longer appear.

Windows won't boot after partition changes? Head to UEFI Repair in the sidebar to rebuild the BCD store or repair the EFI partition.

UEFI Repair

Backup and restore the BCD store, or repair / fully recreate a corrupted EFI System Partition.

Always back up BCD before any repair. Formatting the EFI partition wipes all boot entries — Windows, Linux, and recovery loaders.

Backup BCD Store

Run on a healthy system before making any boot or partition changes. Store the backup on an external drive or cloud.

cmd — admin 
bcdedit /export C:\BCD_Backup\bcdbackup
Verify the file bcdbackup exists at C:\BCD_Backup before closing the prompt.

Restore BCD Store

This overwrites the current BCD configuration entirely. Confirm the backup path is correct before running.
cmd — admin 
bcdedit /import C:\BCD_Backup\bcdbackup

Quick EFI Fix — Repair In Place

Use when: Windows can still boot (or you can access a recovery CMD) but the EFI partition has become corrupted or boot files are missing. This formats the existing EFI partition and rebuilds boot entries without deleting the partition itself.

Reference: youtube.com/CZ17JrgFFhw

1

Backup BCD first

See the Backup BCD Store section above before proceeding.

2

Format the EFI partition and assign a drive letter

The EFI partition is the ~100 MB FAT32 partition. Use list volume to identify it by size and filesystem.

diskpart 
diskpart
list disk
select disk 0         # Replace 0 with the disk containing the EFI partition
list volume           # Find the ~100 MB FAT32 volume
select partition 1    # Replace 1 with the EFI partition number
format fs=fat32 quick
assign letter=V
exit
3

Rebuild Windows boot files

Replace C: if Windows is on a different drive letter.

cmd — admin 
bcdboot C:\Windows /s V: /f UEFI
4

Remove the temporary drive letter and reboot

diskpart 
diskpart
select volume V
remove letter=V
exit

Full EFI Recreate — From Windows Setup

Use when: The EFI partition is missing entirely, or so severely corrupted that the Quick Fix fails. This deletes the old EFI partition and creates a fresh 260 MB one from scratch. Requires booting from a Windows installation USB or DVD.

At the initial Windows Setup screen, press Shift + F10 to open a Command Prompt without proceeding with installation.
1

Identify the existing EFI partition

EFI partition is FAT32, typically between 100 MB and 1 GB.

diskpart 
diskpart
list disk
select disk #          # Replace # with your disk number
list partition         # Find the EFI entry (FAT32, 100–1000 MB)
2

Delete the old EFI partition

diskpart 
select partition #    # Replace # with the EFI partition number
delete partition
3

Create a new 260 MB EFI partition

diskpart 
create partition efi size=260
format quick fs=fat32 label="System"
assign letter=S
exit
4

Rebuild Windows boot files

Replace C:\Windows with your actual Windows path if different.

cmd 
bcdboot C:\Windows /s S: /f UEFI
5

Exit and reboot

Type exit to close the Command Prompt, then remove the installation media and restart.
